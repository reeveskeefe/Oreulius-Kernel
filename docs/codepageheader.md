## Contribution Instructions for New Scripts

- **License and Copyright:**
  - Every new script must begin with the full license and copyright header as shown above.
  - Do not remove or alter the SPDX identifier or copyright.

- **Contribution Guidelines:**
  - By contributing, you agree to license your work under the same terms as the Oreulia project.
  - Follow the [CONTRIBUTING.md](CONTRIBUTING.md) file for code style, review, and submission rules.
  - All public functions, types, and modules must be documented using Rust doc comments (`///`).
  - Use clear, descriptive commit messages for all changes.
  - For major changes, open an issue or discussion before submitting a pull request.
  - Ensure your code passes all tests and builds cleanly before submitting.
  - Respect the project's formatting and linting rules.
  - If you add a new feature, update the feature list and notes in the script header.

- **Header Template Usage:**
  - Copy and paste the entire header block at the top of every new script.
  - Fill in the script/module name and a brief description of its features and purpose.
  - Keep the header up to date as the file evolves.
